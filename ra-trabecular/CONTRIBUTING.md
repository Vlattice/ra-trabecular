# Contributing to RA-Trabecular

Thank you for your interest in contributing to RA-Trabecular. This project is an open scientific framework, and external contributions are welcome.

## Ways to contribute

- **Report issues**: bugs, unclear documentation, suggestions for improvement.
- **Propose new features**: alternative IRCE formulations, additional erosion models, integration with new datasets.
- **Submit code**: bug fixes, performance improvements, new modules.
- **Improve documentation**: tutorials, examples, translations.
- **Validation studies**: applying the framework to new datasets and reporting results.

## Workflow

1. **Open an issue first** for substantial changes — discussion before implementation saves time.
2. **Fork the repository**, create a feature branch from `main`.
3. **Write tests** for new functionality (we use `pytest`).
4. **Follow the code style**: PEP 8 for Python, with type hints where reasonable.
5. **Update CHANGELOG.md** under the `[Unreleased]` section.
6. **Submit a pull request** with a clear description of the change and its motivation.

## Scientific contributions

For contributions that introduce new scientific results (e.g., validation against a new dataset, calibration of IRCE_c, new theoretical extension):

- Provide reproducible code and data references.
- Document the methodology in `docs/`.
- If the contribution merits authorship on a future publication, this can be discussed in the relevant issue.

## Code of conduct

We expect respectful, constructive interaction. Personal attacks, harassment, or dismissive behavior are not tolerated.
